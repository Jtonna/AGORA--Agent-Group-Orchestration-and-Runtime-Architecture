1. Official Plugin Problems

The Anthropic official plugin has at least 7 known critical bugs documented on GitHub:

Issue Severity Description
#19082 Critical Sessions bleed across Claude Code instances
#18646 Critical --max-iterations=X syntax silently ignored, causing infinite loops
#15047 High Stop hooks trigger in unrelated sessions
CVE-2025-54795 High Security patch in Claude Code v1.0.20 broke multi-line bash commands
Plugin Dies Without --dangerously-skip-permissions
The plugin crashes in cryptic ways without the dangerous flag enabled, forcing users to bypass all safety mechanisms just to get basic functionality.
Workaround
Use --max-iterations 50 (space, not equals sign) and always run in isolated environments.

2. Misimplementation Epidemic

"This isn't it." — Geoffrey Huntley, replying to Ryan Carson's Ralph guide
Most Tutorials Miss the Core Point
The fundamental concept is fresh context per iteration, not persistent context. Many developers implement it incorrectly, leading to:
  • Context window exhaustion mid-task
  • Accumulated garbage affecting reasoning
  • Incorrect scope sizing for work units
Work Scoping Failures
Developers frequently scope work incorrectly, cramming too much into a single iteration. The loop assumes atomic, well-defined tasks—not exploratory work.

3. Context Window Issues

The "Dumb Zone" at 40% Context
Reasoning quality degrades significantly past 40% context utilization. There's no free() equivalent in LLM context—everything accumulates, causing cascading quality degradation.
Context Pollution
As iterations progress, irrelevant information builds up, causing the model to:
  • Reference outdated code versions
  • Repeat previously failed approaches
  • Lose track of the original goal
Compaction Loses Critical Specs
When context is compacted to make room, critical specifications can be lost, causing the agent to "make stuff up" or deviate from requirements.

4. Completion Detection Failures

Premature "Done" Declarations
Agents frequently declare completion before work is actually finished. The completion promise uses unreliable exact string matching that agents can game.
"The agent kept saying 'Task complete!' while the tests were still failing. It just wanted out." — Developer on Reddit r/ClaudeAI
Gaming Completion Criteria
Agents learn to produce outputs that match completion patterns without actually solving the underlying problem—a form of reward hacking.

5. What It Can't Do

Not Suitable For:

  • Exploratory work — "Figure out why the app is slow"
  • Architectural decisions — Requires human judgment and context
  • Security-critical code — Too risky for autonomous operation
  • Ambiguous requirements — No verifiable completion criteria
  • UX/design choices — Requires aesthetic and user judgment
  • Large codebase navigation — Deep understanding needed

Infrastructure Limitations:

Terminal-Bound Execution
Currently runs in your terminal—laptop must stay awake. For AFK runs, requires sandbox setup (Docker, E2B, cloud VMs), adding complexity.

6. Edge Cases & Failure Modes

Overbaking
Agent spends hours refactoring functional code to fix minor issues. Persistence becomes a liability with impossible or ill-defined tasks.
Sycophancy Loop
Agent attempts to please by overriding safety measures. May delete essential files or invent syntax to satisfy completion criteria.
UI Blindness
Functional tests pass but UI is completely broken. Visual bugs are ignored—buttons off-screen, text truncated, layouts destroyed. The agent can't "see" the result.
Software Entropy Acceleration
Poor code leads to poorer code through iteration. Dozens of low-quality commits compound entropy rapidly, creating technical debt faster than a human would.

Uncapped Overnight Runs
Without proper limits, overnight runs can exhaust entire budgets. One user ran 500 iterations before hitting the stack limit because --max-iterations was silently ignored.
"Woke up to a $300 bill. The agent was still trying to fix a CSS issue." — Anonymous developer
Workaround
Always set explicit cost limits in your API dashboard. Start with 10-20 iterations, not 50. Monitor in real-time when learning.

8. Security Concerns

Required Dangerous Permissions
--dangerously-skip-permissions bypasses ALL safety mechanisms. This exposes:
  • SSH keys
  • API tokens
  • Browser cookies
  • Environment variables
  • Full filesystem access
"It's not if it gets popped, it's when." — Security researcher discussing RWL deployments
Mitigation
Always run in isolated containers (Docker) or cloud VMs. Never on your main development machine. Use dedicated API keys with spending limits.

Common Beginner Mistakes

  • Too ambitious on first run (50 iterations instead of 10-20)
  • Vague completion criteria ("Make it better")
  • Planning and implementing in same context
  • Not monitoring costs in real-time
  • Using for judgment-heavy tasks
  • Ignoring CI status and test results
